# 認証

パスキー（WebAuthn）によるパスワードレス認証。

## 概要

- パスワード不要
- 生体認証（指紋・顔認証）またはPINで認証
- デバイスに保存されたパスキーを使用

## ユーザー登録

### 前提条件

- パスキー対応ブラウザを使用していること
- 未登録のユーザー名であること

### フロー

1. ユーザー名を入力
2. デバイスのパスキー登録プロンプトが表示
3. 生体認証またはPINで確認
4. 登録完了

### エラーケース

| 状況 | メッセージ |
|------|-----------|
| ユーザー名が既に使用済み | 登録エラー |
| パスキー登録をキャンセル | 認証がキャンセルされました |
| チャレンジの有効期限切れ（5分） | 再度登録をやり直す必要あり |

## ログイン

### 前提条件

- 登録済みのユーザー名であること
- 登録時に使用したデバイス（またはパスキー同期済みデバイス）であること

### フロー

1. ユーザー名を入力
2. デバイスのパスキー認証プロンプトが表示
3. 生体認証またはPINで確認
4. ログイン完了 → ホーム画面へ遷移

### エラーケース

| 状況 | メッセージ |
|------|-----------|
| ユーザー名が未登録 | ログインに失敗しました |
| パスキー認証をキャンセル | 認証がキャンセルされました |
| チャレンジの有効期限切れ（5分） | 再度ログインをやり直す必要あり |

## セッション管理

- ログイン成功時にJWTアクセストークンを発行
- トークン有効期限: **15分**
- トークンはブラウザのローカルストレージに保存
- **スライディング有効期限方式**: 有効期限内のリクエスト毎にトークンを再発行
  - バックエンドのミドルウェアがレスポンスヘッダー `X-New-Token` に新トークンを付与
  - フロントエンドがレスポンスから読み取りローカルストレージを更新
  - 15分間操作がなければトークン失効 → 再ログインが必要

## API

| エンドポイント | 説明 |
|---------------|------|
| `GET /me` | 現在のログインユーザー情報を返却 |

## 認証が必要な操作

- ログイン後のすべての画面・操作にアクセストークンが必要
- トークンが無効または期限切れの場合、APIは `401 Unauthorized` を返却

## 制約・注意事項

- パスキー未対応ブラウザでは利用不可
- 1ユーザーにつき複数のパスキーを登録可能
- パスキー紛失時の復旧手段は未実装
