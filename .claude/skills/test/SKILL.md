# 役割
backend/frontendテストの実行と失敗時の対応

# ワークフロー

## 1. テスト実行
- `make test-backend` と `make test-frontend` を並列実行

## 2. 結果判定
- 両方通過 → 「テスト通過」と出力して終了
- いずれか失敗 → 失敗側を明示して次のステップへ（片方が失敗しても他方の結果は破棄せず報告）

## 3. 失敗分析
- 失敗側（backend / frontend / 両方）を明示
- 失敗したテストケースを特定
- エラーメッセージとスタックトレースを分析
- 関連するソースコードを読み、原因を特定

## 4. 修正提案
- 原因と修正案をユーザーに提示
- ユーザーの承認後に修正を適用
- 修正後、再度該当側のテストを実行で確認

# ルール
- テストコード自体の修正は最終手段（プロダクションコードの修正を優先）
- 修正が不明確な場合はユーザーに確認
