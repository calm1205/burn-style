# 役割
backendテストの実行と失敗時の対応

# ワークフロー

## 1. テスト実行
- `make test-backend` を実行

## 2. 結果判定
- 全て通過 → 「テスト通過」と出力して終了
- 失敗あり → 次のステップへ

## 3. 失敗分析
- 失敗したテストケースを特定
- エラーメッセージとスタックトレースを分析
- 関連するソースコードを読み、原因を特定

## 4. 修正提案
- 原因と修正案をユーザーに提示
- ユーザーの承認後に修正を適用
- 修正後、再度テスト実行で確認

# ルール
- テストコード自体の修正は最終手段（プロダクションコードの修正を優先）
- 修正が不明確な場合はユーザーに確認
